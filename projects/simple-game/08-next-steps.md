## Learn C++

When you're comfortable with Blueprints, learn C++:

### Why C++

- Better performance
- More control
- Industry standard
- Required for engine modification

### Learning Path

1. **Basic C++** - Variables, functions, classes
2. **Pointers and Memory** - Critical for game dev
3. **Unreal C++ Specifics** - UCLASS, UPROPERTY, etc.
4. **Convert Blueprints to C++** - Great practice

### Resources

- [learncpp.com](https://www.learncpp.com) - Free comprehensive tutorial
- "Beginning C++ Through Game Programming" book
- Unreal's C++ programming tutorials

## Career Paths

If you want to work in games:

### Roles

- **Gameplay Programmer** - Game mechanics
- **Engine Programmer** - Low-level systems
- **Technical Artist** - Bridge art and code
- **Tools Programmer** - Editor and pipeline tools
- **Level Designer** - Using engine to build content

### Portfolio Tips

- Quality over quantity
- Show complete projects
- Explain your contributions
- Include source code (if possible)
- Show problem-solving

### Getting Experience

- Contribute to open source
- Make mods for existing games
- Join indie teams
- Create your own games
- Build a portfolio

## Final Thoughts

### Remember

1. **Everyone starts somewhere** - Don't compare yourself to experts
2. **Finish projects** - A finished simple game beats an unfinished dream
3. **Learn by doing** - Tutorials help, but making things teaches more
4. **Ask for help** - The community is generally helpful
5. **Have fun** - If you're not enjoying it, take a break

### The Most Important Thing

**Make games.**

Not "learn about making games." Not "plan to make games." Not "watch tutorials about making games."

Actually make them. Start small. Finish them. Then make the next one.

Each game teaches you something. Each failure teaches you more.
